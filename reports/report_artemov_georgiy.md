## 2. Перечень задач и описание работы над ними

### 2.1. Реализация модели групп

**Задача:** Создать модель групп для хранения информации о группах.

**Описание работы:**

На данном этапе была реализована модель групп - `groups` в БД.

Таблица имеет следующие поля:
- идентификатор группы,
- наименование группы,
- идентификатор владельца группы.

Связь с таблицей пользователей, осуществляется через соединительное хранилище `group_members`.

### 2.2. Реализация модели участников групп

**Задача:** Создать модель групп для хранения информации об участниках групп.

**Описание работы:**

Таблица представляет собой связки пар ключей пользователя и группы, в которой он является ее участником:
- идентификатор пользователя,
- идентификатор группы.

### 2.3. Реализация CRUD-функциональности для групп и ее участников

**Задача:**  
Реализовать API-эндпоинты для управления группой и ее составом.

**Описание работы**

Были осуществлены следующие операции:
- `GET /group/{group_id}` - получение информации о группе
- `POST /group/create` - создание группы
- `PUT /group/{group_id}/update` - обновление группы
- `DELETE /group/{group_id}` - удаление группы
- `POST /group-members/add_member` - добавление участника в группу
- `DELETE /group-members/delete` - удаление участника из группы

В итоге пользователь может создавать группы и управлять их составом (добавляя/удаляя участников)

### 2.4. Добавление аналитики для групп

**Задача:**  
Реализовать аналитику для групп пользователей.

**Описание работы**

Была добавлена возможность относить транзакции к группам. Благодаря этому, стало возможным анализировать расходы групп.
Для достижения этой цели, дополнительно изучен SQL, для составления правильных запросов к БД.
